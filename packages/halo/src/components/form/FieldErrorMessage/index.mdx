# FieldErrorMessage

Prints a supplied `message` styled as a warning or critical error. Typically, component directly follows an input component.

Preset `block` is great for rendering a critical server error message at the header or footer of a given form.

### Usage

<Playground>
  <div className="max-w-xl">
    <FieldErrorMessage errorSeverity="critical" preset="block" message="Critical server error. Please try again." />

    <FieldErrorMessage errorSeverity="warning" preset="block" message="Relatively soft form warning" />

    <Label field="name" title="Field warning" byline="ex: validation error" className="mt-10">
      <Input
        name="name"
        onChange={() => {}}
        type="text"
        value="bad name"
        hasError
      />
      <FieldErrorMessage errorSeverity="warning" message="A good name is required" />
    </Label>

    <Label field="address" title="Field critical error">
      <Input
        name="address"
        onChange={() => {}}
        type="text"
        value="123 Wewtlane"
        errorSeverity="critical"
        hasError
      />
      <FieldErrorMessage errorSeverity="critical" message="City, state, and zip code required" />
    </Label>
  </div>
</Playground>


### Exports

```ts
type FieldErrorMessagePreset = 'text-only' | 'block';

type FieldErrorMessageSeverity = 'warning' | 'critical';
```

### Props

<Props of={FieldErrorMessage} />


### Example with Formik

```jsx
import { ErrorMessage } from 'formik';

<ErrorMessage
  name="salaryMax"
  render={message => <FieldErrorMessage message={message} />}
/>
```
